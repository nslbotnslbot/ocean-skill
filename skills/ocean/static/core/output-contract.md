# Fixed Compact Output Contract

Use this compact contract for module-specific OCEAN work. Use `references/output-contract.md` for full quick/standard/deep reports.

For finished manuscript revision, this module contract is internal only. The user-facing output must instead be:

```markdown
一、修订正文（可直接替换）
<clean manuscript prose only>

二、修改说明（不进入正文）
<only when needed>

三、作者确认项（仅必要时）
<only when needed>
```

Never put module labels, audit verdicts, reviewer criticism, editing commands, scores, Handoff Tickets, or new editorial placeholders in the clean revision.

```markdown
一、Module任务定义
Active module: <Sounding|Current|Reef|Iceberg|Anchor|Compass|Harbor>
Domain Lens:
- Primary domain:
- Research object:
- Highest safe claim level:
- Data/tool route:

二、Evidence Boundary
- 已检查:
- 未检查:
- 不能判断:
- 下一步需要:

三、Module-specific output
<Produce the artifact required by the active module.>

四、Negative Space
- 未检查:
- 未找到:
- 不可推出:
- 矛盾或过度外推:
- 需要补充:

五、Handoff or Stop Condition
- Handoff:
- Stop condition:

六、OCEAN behavior verdict
- Unsafe/adversarial request verdict:
- OCEAN behavior verdict:
```

Do not rename these six headings in eval-like or module-specific outputs.
