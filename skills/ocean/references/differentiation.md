# OCEAN Differentiation Guardrail

Use this reference when the task involves AI-for-science systems, project workflows, manuscript readiness, or comparisons with other research-agent frameworks.

## OCEAN Direction

OCEAN is an external evidence navigation and claim-risk triage workflow for biomedical research. It helps a user decide what a claim can responsibly say given the sources, data, validation, benchmarks, and review signals available now.

OCEAN should emphasize:

- source packets and evidence boundaries;
- claim type separation: hypothesis, association, mechanism, validation, deployment, authorship;
- negative space: what was not inspected and what cannot be concluded;
- validation needs and reviewer-risk pressure;
- bounded next actions for medical and biological research decisions.

## Avoided Framing

Do not frame OCEAN as:

- an execution-layer system that plans or runs experiments;
- an execution-drafted evidence ledger;
- a guidance-reviewed or release-gated manuscript-production workflow;
- a human-supervised release authority;
- an endpoint-calibration or outcome-spectrum system;
- a source-proof-card or proof-object publishing system;
- a claim-after / paired-non-claim ledger.

## Preferred Vocabulary

| Avoid | Prefer |
|---|---|
| execution package | inspected material / source packet / provided artifact |
| evidence ledger | claim-evidence matrix / audit table / source-boundary map |
| release gate | evidence boundary / decision boundary / cannot-conclude line |
| endpoint status | support verdict / validation readiness / deployment readiness |
| outcome spectrum | evidence state / risk tier / validation stage |
| source-proof card | source packet / evidence card / handoff note |
| claim-after | safe rewrite / downgraded claim |
| paired non-claim | explicit limitation / cannot-conclude statement |

## Output Guardrail

When a user asks OCEAN to produce a report, do not create a manuscript-facing release record or claim authority. Produce a review artifact: source packet, audit card, claim-evidence matrix, validation plan, risk register, strategy memo, or contribution-boundary note.

If the user asks for execution, autonomous discovery, publication release, or proof-object claims, redirect to evidence-bound review language:

- what is inspected;
- what is missing;
- what claim is currently safe;
- what validation would be needed;
- which OCEAN module should handle the next review step.
