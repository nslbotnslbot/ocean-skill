# Claim-to-Artifact Matrix

Use this matrix when the user provides code, figures, data files, logs, manuscripts, database/API outputs, reviewer comments, or project notes.

## Matrix

| Claim ID | Claim | Claim type | Required support | Artifact/source provided | Inspected? | Support verdict | Missing artifact | Safe rewrite / decision |
|---|---|---|---|---|---|---|---|---|

## Claim types

- descriptive
- method
- performance
- benchmark
- mechanism
- causal
- clinical
- publication/status
- contribution/authorship
- trend/field-positioning

## Support verdicts

- Supported
- Partially supported
- Downgrade
- Unsupported
- Contradicted
- Cannot judge
- Needs source

## Required support examples

- Performance claim: metric table, benchmark protocol, split definition, confidence intervals, leakage audit.
- Mechanism claim: perturbation/assay/causal design, not database association alone.
- Clinical claim: external/prospective/implementation evidence, calibration, utility, safety.
- Reproducibility claim: data, code, environment, parameters, seed, execution log, output lineage.
- Authorship/status claim: explicit written record, not meeting implication.
- Trend claim: search corpus, query, date range, inclusion criteria, inspected sources.
