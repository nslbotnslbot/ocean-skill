# OCEAN Application and Submission Tracker

This page tracks research projects that use **OCEAN: Orchestrated Claim-Evidence Analysis Navigator** in real manuscript planning, claim-evidence audit, validation planning, submission preparation, or revision workflows.

## Context

**中文上下文**: 本页用于记录已经在真实研究或论文投稿流程中使用 OCEAN 的案例。它展示 OCEAN 不只是模板或提示词集合，而是可以进入研究设计、证据审计、投稿前点检和修稿判断的实践工作流。

**English context**: This page records real research or manuscript workflows that use OCEAN. It is intended to show OCEAN as a runnable, auditable, and transferable research workflow rather than a static template collection.

**Scope / 影响范围**: Public project-level tracker only. This page does not publish manuscripts, private peer-review text, patient-level data, unpublished raw data, journal correspondence, or confidential collaborator notes.

**Evidence boundary / 证据边界**: The existence of the two application cases below is based on user-provided project information. Exact OCEAN artifacts, submission dates, journal names, review outcomes, and manuscript decisions must be updated only after they are confirmed by the project owner.

## Status Vocabulary

Use these labels consistently when updating the tracker.

| Status label | Meaning |
|---|---|
| OCEAN planning | OCEAN is being used for research design, source packet construction, or project route selection. |
| OCEAN pre-submission audit | OCEAN is being used for claim-evidence audit, validation planning, or submission-readiness checks before journal submission. |
| Submitted | Manuscript has been submitted to a journal or platform. |
| Under review | Manuscript is in peer review. |
| Revision | Manuscript is being revised after editorial or reviewer feedback. |
| Accepted | Manuscript has been accepted. |
| Published | Manuscript is publicly available as a publication or preprint. |
| Paused | Submission or revision is temporarily paused. |
| To be updated | Current status has not yet been confirmed for public reporting. |

## Public Process Log Template

Each case can later receive a dated process entry using the following public-safe structure.

| Field | Public-safe content to record |
|---|---|
| Date | Date of the OCEAN run or submission-stage update. |
| Manuscript stage | Planning, pre-submission audit, submitted, under review, revision, accepted, or published. |
| OCEAN modules used | Sounding, Current, Reef, Iceberg, Anchor, Compass, Harbor, or a subset. |
| Decision supported | For example: claim downgrade, validation-plan revision, journal-fit decision, response strategy, or next experiment. |
| Evidence boundary | What was inspected, what was not inspected, what cannot be concluded, and what must be checked next. |
| Public artifact | Link to public-safe source packet, claim-evidence table, validation checklist, Harbor memo, or summary report. |
| Confidentiality note | Explicitly state what is excluded from public release. |

## Reuse Note

This tracker can be reused as evidence that OCEAN has entered real manuscript workflows, but it must not be reused as evidence that a manuscript has been accepted, validated, externally confirmed, or clinically useful unless those facts are separately documented and publicly confirmable.
